Educational Reference
What is a CFG? ▼
A Context-Free Grammar (CFG) is a formal grammar G = (V, Σ, R, S) where:

• V — Non-terminals (e.g. S, A, B)
• Σ — Terminals (e.g. a, b, +, *)
• R — Production rules (e.g. S → aSb)
• S — Start symbol

CFGs generate context-free languages and are fundamental to programming language design and parsing theory.
Leftmost vs Rightmost Derivation ▼
Leftmost Derivation: Always expand the leftmost non-terminal first.
S ⇒ aSb ⇒ aaSbb ⇒ aabb

Rightmost Derivation: Always expand the rightmost non-terminal first.
S ⇒ aSb ⇒ aεb ⇒ ab

Both produce the same language but trace different paths. They're used to construct LL and LR parsers respectively.
Grammar Ambiguity ▼
A grammar is ambiguous if there exists a string with two or more distinct parse trees (i.e., two leftmost derivations).

Ambiguity is problematic in compilers because the meaning of a program depends on its parse tree. For example, 3+4*2 could be parsed as (3+4)*2=14 or 3+(4*2)=11 in an ambiguous arithmetic grammar.
Parse Tree Structure ▼
A parse tree visually shows how a string is derived:

• Root — Start symbol (S)
• Internal nodes — Non-terminals
• Leaves — Terminals or ε

The frontier (leaves read left-to-right) gives the derived string.
